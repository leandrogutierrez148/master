Podemos asumir que un mapeo no homogeneo de las claves produjeron la inconsistencia en el conteo. Adems observando podemos intuir el problema: \[ \begin{aligned} Arab\ Rep.\ Egypt &= Egypt,\ Arab Rep. \\ Rep.\ Yemen &= Yemen,\ Rep. \\ Russian &= Russian\ Federation \end{aligned} \]

Debemos codificar de manera que nuestras claves estn etiquetadas de manera uniforme y cumplir con los requerimientos iniciales. Para ello crearemos una columna auxiliar donde estar el nuevo valor codificado de la variable pais

# declaramos nuestro funcion de mapeo
mapear <- function(pais) {
    if (pais == 'Egypt, Arab Rep.'){
        return('Arab Rep. Egypt')
    } else if(pais == 'Yemen, Rep.'){
        return('Rep. Yemen')
    }else if(pais == 'Russian'){
        return('Russian Federation')
    }
    return(pais)
}

# creamos una nueva columna con el resultado de aplicar la funcion `mapear` a cada elemento de la columna `pais`
df_aux$pais_r <- sapply(as.vector(df_aux$pais), mapear)

  # intentamos convertir a numeric

Respectivamente, para poder ser casteables a numeric sin problemas.

Crearemos una funcin limpiar_numeric para eliminar el caracter . (punto) y luego reemplazar , (coma) por ., quedando un valor tipo char casteable a numeric.

# declaramos la funcion
limpiar_numeric <- function(x) {
    # eliminamos puntos
    gsub("\\.", "", x) %>%
    # reemplazamos coma por punto
    gsub(",", ".", .)
}

# aplicamos la funcion limpiar a todos los valores del campo poblacion
